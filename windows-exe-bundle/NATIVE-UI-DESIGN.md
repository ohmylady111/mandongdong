# Native Desktop UI 方案（不调用浏览器）

目标：把当前基于本地 HTTP + 浏览器的界面，改成 **真正的原生桌面 GUI**。

## 推荐技术选型

**首选：PySide6 / Qt for Python**

原因：
- 原生桌面窗口，不需要浏览器
- Windows 打包成 EXE 成熟
- 表单、日志、进度条、列表、文件选择器都好做
- 后续可加托盘、设置页、主题、图标

不推荐继续用浏览器的原因：
- 公司电脑有时会拦截本地服务 / 默认浏览器策略
- 双击后“像网页”而不像软件
- 调试成本高于原生 UI

## B. 左侧：配置面板

### 基本信息
- URL
- 输出目录
- 标题（可空，自动取）
- Referer（可空）
- User-Agent（高级，可折叠）

### 选择器设置
- 图片选择器（多行）
- 章节/子页选择器（多行）
- 图片过滤正则
- 页面过滤正则

### 下载参数
- dynamic（复选框）
- wait_ms
- delay
- retries
- timeout
- same_origin_only（复选框）
- flat（复选框）
- skip_existing（复选框）
- save_discovery（复选框）

### 抓取范围
- page_start
- page_end
- page_limit

### 配置操作
- 保存为模板
- 另存为模板
- 删除模板
- 从模板加载

---

## C. 右侧：结果区（Tab）

建议做成三个标签页：

### Tab 1：实时日志
- 滚动日志窗口
- 不同颜色显示：info / warn / error / success
- 支持复制
- 支持保存日志

### Tab 2：预览结果
Dry Run 后展示：
- 发现多少页面
- 每页抓到多少图片
- 选中的图片 URL 列表
- 可点开查看前几张缩略图（可后做）

### Tab 3：历史任务
- 时间
- 标题
- URL
- 输出目录
- 状态
- 打开 manifest
- 重新运行

---

## D. 底部状态栏

显示：
- 当前进度（进度条）
- 已扫描页数 / 总页数
- 已下载图片数
- 已跳过数
- 失败数
- 当前保存路径

## 操作流程设计

### 流程 1：用户首次使用
1. 打开程序
2. 粘贴 URL
3. 选择输出目录
4. 填图片选择器
5. 点击“预览（Dry Run）”
6. 看右侧预览结果
7. 没问题后点击“开始下载”

### 流程 2：用户长期复用
1. 打开程序
2. 直接从模板列表选择某个站点模板
3. 改一下 URL / 标题
4. 点击开始下载

### 流程 3：出错处理
- 弹窗显示错误摘要
- 右侧日志保留完整错误
- 提供“复制错误”按钮

## 实现建议（技术）

### UI 层
- PySide6
- QMainWindow
- QSplitter
- QTabWidget
- QTextEdit / QPlainTextEdit（日志）
- QFileDialog（目录选择）
- QThread / QRunnable（后台任务）

### 后台任务层
- 保留现有下载逻辑
- 把下载逻辑放进 worker thread
- 用 signal/slot 回传：
  - 日志
  - 进度
  - 完成状态
  - 错误

### 配置层
- JSON 模板
- 最近使用配置
- 历史任务记录

---

## 最小可交付版本（MVP）

第一版我建议先做这些：
- 原生窗口
- URL / 输出目录 / 选择器输入
- Dry Run 按钮
- 开始下载 / 停止下载
- 实时日志
- 保存 / 载入配置
- 简单进度条

先不急着做：
- 缩略图预览
- 拖拽导入
- 托盘
- 多任务并发
- 插件系统

---

## 结论

如果目标是：
- 公司电脑上更稳定
- 双击像软件
- 不依赖浏览器
- 后续更容易做成正式 EXE / 安装包

那就应该从现在这个“浏览器 UI”切到：

**PySide6 原生桌面 GUI 版**
